# Zelyo Implementation Plan — Index & Interface Contract

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement these plans task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build Zelyo — ZK-backed verifiable credentials on Stellar: an issuer mints a credential to a learner's wallet; the learner proves one fact in zero-knowledge in-browser; a Soroban contract verifies and records only a nullifier.

**Architecture:** pnpm monorepo. A Noir circuit proves Merkle inclusion + nullifier + address binding + one disclosed predicate. A Soroban `CredentialRegistry` contract enforces root validity and nullifier uniqueness (verifying the proof on-chain on Path A, or recording a server-verified proof on Path B — see design doc §2). A Next.js 16 app hosts the issuer portal, holder wallet (with in-browser proving), and public verifier/job-board. Postgres mirrors chain state for UX; the chain is the source of truth.

**Tech Stack:** Node 22 · pnpm 10 · Next.js 16.2 (App Router) · React 19.2 · TypeScript 6 (strict) · Tailwind v4.3 · Prisma 7.8 · Postgres 16 · Auth.js v5 · Zod 4 · Noir 1.0.0-beta.22 · `@aztec/bb.js` 4.3 · `@stellar/stellar-sdk` 16 · Rust `soroban-sdk` · Vitest 4 · Playwright 1.

> Source documents: `SPEC.md` (build contract), `BRAND.md` (design system), `AGENT.md` (engineering rules), `docs/superpowers/specs/2026-06-23-zelyo-build-design.md` (decisions & decomposition). When SPEC and AGENT disagree on security, the stricter rule wins.

- Verify each with `pnpm view <pkg> version` before pinning.

**TypeScript:** `strict: true`, `noUncheckedIndexedAccess`, `exactOptionalPropertyTypes`. No `any`; use `unknown` + Zod at boundaries.

**Security (mandatory, from AGENT.md §4):**
- Personal data (name/grade/attributes) **never** goes on-chain, in logs, in analytics, or in client bundles.
- Holder secret `s` is generated with WebCrypto in the browser, stored encrypted in IndexedDB + user-exportable backup, and **never transmitted**. Server only ever sees `id_commitment = Poseidon(s)` and nullifiers. Server rejects any payload that would contain `s`.
- `ISSUER_SECRET` and any signing key are server-only, from env, never bundled, never `NEXT_PUBLIC_`, never logged.
- Passwords: argon2id via `@node-rs/argon2`. Never log/store plaintext. Generic auth errors ("invalid credentials"); constant-time delay + rate limit on login.
- Every external input Zod-validated and size-bounded. Prisma only (no raw SQL interpolation).
- Route handlers are thin: validate → authorize → call a `src/server/*` service → map to typed response. Business logic in services.
- Errors: throw typed `AppError(code, httpStatus, publicMessage)`; one error boundary maps to `{ error: { code, message } }`. Never leak stack traces or DB errors.
- RBAC enforced in middleware **and again in every handler**: `ADMIN` for `/issuer/**` and `/admin/**`, `HOLDER` for `/wallet/**`. Never trust client-sent role.
- Logging: structured `pino`; redact `authorization`, `password`, `set-cookie`, `s`, attributes. Log a request id.
- Security headers in `next.config.ts`: CSP (no `unsafe-inline` scripts), `X-Frame-Options: DENY`, `X-Content-Type-Options: nosniff`, `Referrer-Policy: strict-origin-when-cross-origin`, minimal `Permissions-Policy`. **`Cross-Origin-Opener-Policy: same-origin` + `Cross-Origin-Embedder-Policy: require-corp`** on the app and prover/artifact routes (bb.js WASM threads need cross-origin isolation, else proving silently fails).
- Rate limits (`rate-limiter-flexible` + Redis, per IP): auth 10/min, `/api/verify` 20/min, register 5/min, mint 60/min. Return `429` with `Retry-After`.
- Private S3 bucket; serve VC files via short-lived signed URLs. Audit-log mint/revoke/verify to `AuditLog` with actor + ip (no PII values).

**Correctness-critical (from AGENT.md §5):**
- **Poseidon parity:** JS leaf/nullifier builders in `packages/zk-shared` must output the exact field elements the Noir circuit computes (same Poseidon params over BN254). A parity test with fixed vectors fails CI on drift.
- `scope` must match between client (`ZK_SCOPE_APP_ID` + chain id + registry id) and contract. Bind proof to `bound_address`; contract asserts it equals the invoker.
- Duplicate nullifier reverts (`NullifierUsed`) — that revert **is** the Sybil block; surface it cleanly in the UI.

**Conventions:** Conventional Commits. Server-only modules carry `import "server-only";`. Parse `process.env` once through a typed Zod schema in `src/lib/env.ts`; fail fast on boot. New env vars go in `.env.example` **and** `env.ts`. Brand: one foil-stamp CTA per view; match `BRAND.md` tokens exactly; respect `prefers-reduced-motion`.

**Definition of done (every task):** builds; `typecheck`, `lint`, relevant unit/e2e pass; inputs Zod-validated; mutating routes RBAC-guarded + rate-limited; no secret/PII logged or shipped to client; new env vars in `.env.example` + `env.ts`; visibly matches `BRAND.md`. ZK/contract changes: Poseidon parity test + duplicate-nullifier revert test pass.

## Path A / Path B (the spike decision)

Phase 0's spike sets `ZK_VERIFY_MODE` = `onchain` (Path A) or `server` (Path B). All downstream phases program against the **`verifyAndRegister` service interface** below; only its implementation differs. Phase 5/6 UI and acceptance criteria are identical under both paths.

## Cross-Phase Interface Contract

> Every phase plan MUST use these exact names and signatures in its `Interfaces:` blocks. This section is the single source of truth for cross-phase types.

### `packages/zk-shared` (Phase 1)

```ts
// Field element as 0x-prefixed lowercase hex, 32 bytes. Branded for safety.
export type FieldHex = string & { readonly __brand: "FieldHex" };

export interface Attributes {
  track: string;        // disclosed predicate target
  grade: string;
  issueDate: string;    // ISO 8601
  courseName: string;
  learnerName: string;
}

export interface PublicInputs {
  root: FieldHex;
  scope: FieldHex;
  boundAddress: FieldHex;  // Stellar ed25519 pubkey, field-packed
  nullifier: FieldHex;
  disclosed: FieldHex;     // hash/encoding of the revealed attribute (track)
}

export interface ProofBundle {
  proof: Uint8Array;
  publicInputs: PublicInputs;
}

// Poseidon over BN254 — MUST match circuit params (parity-tested).
export function poseidon(inputs: FieldHex[]): FieldHex;
export function idCommitment(s: FieldHex): FieldHex;                 // Poseidon(s)
export function buildLeaf(idCommitment: FieldHex, attributes: Attributes): FieldHex; // Poseidon(idCommitment, Poseidon(attributes))
export function computeNullifier(s: FieldHex, scope: FieldHex): FieldHex;            // Poseidon(s, scope)
export function computeScope(appId: string, chainId: string, registryId: string): FieldHex; // Hash(app_id|chain_id|registry_id)
export function encodeAddressToField(stellarPubKey: string): FieldHex;              // G... -> field-packed
export const MERKLE_DEPTH = 20;
```

### Soroban `CredentialRegistry` (Phase 2)

```rust
// Errors
pub enum Error { NotAuthorized = 1, UnknownRoot = 2, NullifierUsed = 3, AddressMismatch = 4, InvalidProof = 5 }

set_root(env, issuer: Address, root: BytesN<32>)                 // issuer-only; appends to valid root set
verify_and_register(env, proof: Bytes, pi: PublicInputsXdr) -> () // Path A: verify proof on-chain, then checks below
register(env, pi: PublicInputsXdr, attestor: Address) -> ()       // Path B: server-attested; same checks minus on-chain verify
is_root_valid(env, root: BytesN<32>) -> bool
is_nullifier_used(env, nullifier: BytesN<32>) -> bool
revoke_root(env, issuer: Address, root: BytesN<32>)               // Could: invalidates old root
// Checks (both paths): root ∈ valid set → boundAddress == invoker → nullifier unused → store nullifier → emit Verified(nullifier, root, boundAddress)
```
